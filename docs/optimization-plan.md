## 📝 详细方案

### P0-1: API 封装

**问题：**
- 当前 API 直接使用 fetch，无统一拦截器
- 无统一错误处理
- 无请求 Loading 状态管理

**目标结构：**
```
api/
├── index.ts           # axios 实例 + 拦截器
├── interceptor.ts    # 响应拦截器
└── modules/
    ├── productOverview.ts
    └── user.ts
```

**实现内容：**
- [ ] 创建 axios 实例，统一 baseURL
- [ ] 请求拦截器：Token 自动注入
- [ ] 响应拦截器：错误统一处理
- [ ] 按模块划分 API 文件

---

### P0-2: 路由守卫

**问题：**
- 无权限验证
- 无登录状态校验
- 路由变化无记录

**目标结构：**
```
router/
├── index.ts           # 路由主入口
├── guard/
│   ├── index.ts       # 守卫主入口
│   ├── permission.ts  # 权限守卫
│   └── userLoginInfo.ts
└── routes/
    └── index.ts
```

**实现内容：**
- [ ] 登录状态验证
- [ ] 权限校验
- [ ] 路由变化事件
- [ ] 404/403 错误处理

---

### P0-3: 类型定义完善

**当前问题：**
- DomainItem 类型分散
- 无统一类型定义文件
- Props 类型定义不完整

**目标结构：**
```
types/
├── index.ts           # 统一导出
├── domain.ts          # 域类型
├── productOverview.ts  # 产品全景类型
└── api.ts             # API 响应类型
```

**实现内容：**
- [ ] 统一类型定义文件
- [ ] API 响应类型
- [ ] 组件 Props 类型
- [ ] 事件类型定义
